# 2026-PDS-Elephant

This Data Science Project aims to build a system that classifies skin lesions as
**cancerous** or **non-cancerous** from clinical images taken by smartphones. 
We use handcrafted visual features and an interpretable Decision Tree classifier.

Two pipelines are compared with one another:

- **Baseline** that extracts features after contrast enhancement (CLAHE)
- **Extended** that removes hair before extraction and adds a hair-coverage feature

The guiding question is **"Does removing hair as a preprocessing step
improve classification performance or not?"**

## Dataset

We use the [PAD-UFES-20](https://www.sciencedirect.com/science/article/pii/S235234092031115X) dataset with associated segmentation masks provided by the Project in Data Science course. 
Images are grouped into a binary label:

- **Cancerous (1):** BCC, SCC, MEL
- **Non-cancerous (0):** ACK, NEV, SEK

The dataset images and masks are **not** included in this repository as instructed (see
`.gitignore`). To run the project, place the images in `data/imgs/` and the
masks in `data/masks/`.

## Setup

Requires Python 3. Install dependencies:

```bash
pip install -r requirements.txt
```

## How to run

### 1. Extract features

From inside the `src/` directory, run each extraction script. This reads the
images and masks from `data/` and writes a feature table to `data/`.

```bash
cd src
python extract_features_baseline.py
python extract_features_extended.py
```

This produces `data/features_baseline.csv` and `data/features_extended.csv`.

### 2. Train and evaluate

From the repository root, run the two training scripts. Each performs a
patient-level train/test split, selects the best tree depth via GroupKFold
cross-validation, trains the model, saves it to `results/models/` and writes
predictions and metrics to `results/`.

```bash
python main_baseline.py
python main_extended.py
```

### 3. Load a saved model and predict

`main.py` loads a previously saved model and produces predictions without
retraining. Set the configuration variables at the top of the file:

- `load_model = True`
- `mode = "baseline"` or `mode = "extended"`

Then run:

```bash
python main.py
```

This reads the corresponding feature CSV and saved model and writes
predictions to `results/`.

### Open question

`open_question_hair.py` produces the analysis of how much the presence of hair
affects classification results.

```bash
python open_question_hair.py
```

## Method summary

**Features** inspired by the ABC components of the dermatological ABCD rule:

- **Asymmetry:**  XOR mismatch between mirrored halves of the lesion mask,
  averaged over rotations
- **Border (compactness):** perimeter² / (4π · area); higher values indicate
  more irregular shapes
- **Colour:** mean and standard deviation of HSV channels, plus hue entropy
- **Texture:** mean Sobel gradient magnitude inside the lesion
- **Hair coverage:** (extended only) fraction of the image detected as hair

**Preprocessing:** CLAHE contrast enhancement (both pipelines); morphological
hair removal with inpainting (extended pipeline only).

**Classifier:** Decision Tree, with depth chosen by GroupKFold
cross-validation. All splits are performed at the patient level to prevent
data leakage between training and test sets.

**Metrics:** Accuracy, Recall, Precision, F1 and AUC. Recall is prioritised,
since missing a cancerous lesion is more costly than a false alarm in a
screening context.
